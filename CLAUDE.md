# CLAUDE.md — проект «Футбол 98» (Retro soccer)

## Кто заказчик
Олег — вайб-кодер, НЕ программист. Объясняй каждый шаг простым языком: что делает команда, зачем файл, что произойдёт. Термин неизбежен — расшифруй в скобках. Все ответы — на русском.

## Что мы делаем
Браузерный ретро-футбол 11×11 в эстетике конца 90-х: PS1 low-poly, CRT-фильтр, русский комментатор, реальные составы 96–99. Полное видение и план фаз — **CONCEPT.md, прочитай его первым делом.** Журнал сессий — PROGRESS.md.

## Эстетика (железное правило)
Камертон — **телетрансляция футбола 1996–98**: кинескоп, угловатые фигурки, телеграфика с большими цифрами, шум трибун. Каждое решение (модель, цвет, шрифт, звук, UI) проверяй вопросом: **«выглядит ли это как футбол по телевизору конца 90-х?»** Глянцевое/современное/мультяшное — переделать. Шрифты табло: Press Start 2P / Jersey 10 (Google Fonts).

## Геймплей (железное правило)
**Реализм и оборона важнее зрелищности.** Типичный счёт 1:0–3:2; разгром = баг баланса. Каждая правка геймплея проверяется вопросом: «не стали ли голы слишком дешёвыми?» Все константы баланса — ТОЛЬКО в `src/config.js` (наш аналог Params.ini из Simple Soccer), в логике не хардкодить.

## Технологии (НЕ менять без обсуждения)
- Чистый HTML + JS + **Three.js через CDN** (importmap + ES-модули). Без сборщиков и npm.
- Рендер сцены в низком разрешении (~320×240) с апскейлом + **один** объединённый CRT-проход (pmndrs/postprocessing).
- Игроки, стадион, мяч — из примитивов Three.js кодом (стиль PS1). Текстуры — папка `textures/`.
- Звук — Howler.js, файлы **MP3** (OGG не играет в Safari). Комментатор — заранее сгенерированные клипы ElevenLabs.
- Запуск: локальный сервер `python3 -m http.server` (при первом запуске объясни Олегу как).
- Целевые платформы: браузер ПК **и** планшета (iPad Safari, Android Chrome). Любая новая фича обязана работать с тач-управлением и не ронять FPS планшета.

## Архитектура (железные правила)
1. **Данные ≠ код.** Составы — `data/teams/*.json`, тексты комментатора — `data/commentary.json`, баланс — `src/config.js`. Новая команда = новый JSON, без правки логики.
2. **ES-модули с первого дня.** index.html тонкий; логика в `src/` по папкам (ai/, render/, input/...). Один файл-простыня недопустим.
3. **Реальные имена — сменный слой.** Реальные составы живут отдельными файлами данных; переход на PES-имена при релизе = замена файлов, код не трогаем.
4. **AI строго по слоям:** «тренер» (стейт-машина команды) → «голова» (стейт-машина роли игрока) → «ноги» (steering behaviors, каждый кадр). Слои не смешивать. Архитектура — [База-знаний/Исследования/03-Футбольный-AI.md](База-знаний/Исследования/03-Футбольный-AI.md).

## Тестирование

- Раз в фазу — реальный тест на iPad силами Олега. Напоминай про жёсткое обновление страницы: браузер кэширует 404 (грабля из «Новоселья» — файл появился, а браузер помнит, что его не было).
