# CONCEPT — «Футбол 98» (рабочее название, придумаем лучше)

Обновлено: 17 июля 2026. Это главный документ видения — сюда смотрим, когда спорим «а что мы вообще делаем».

## Видение одной фразой

Ты включил телевизор в 1998-м: шумит стадион, картинка чуть плывёт на кинескопе, комментатор кричит «ГО-О-ОЛ!», низкополигональные фигурки разыгрывают мяч — только теперь это играется в браузере на любом устройстве, и защита в этой игре наконец-то умеет защищаться.

## Зафиксированные решения (интервью 17.07.2026)

| Вопрос | Решение |
|---|---|
| Графика | Low-poly 3D в стиле PS1, ТВ-камера сбоку, CRT-фильтр поверх |
| Масштаб | Полные 11 на 11 |
| Команды на старте | 8 сборных эпохи ЧМ-98/Евро-96: Франция, Бразилия, Италия, Германия, Англия, Голландия, Аргентина, Хорватия |
| Соперники | Против AI + вдвоём за одним экраном (hot-seat). Онлайн — в мечты, не делаем |
| Правила | Ауты/угловые/вбрасывания + офсайды + фолы/карточки/штрафные + усталость и замены (вводим поэтапно) |
| Комментатор | Русский, стиль телекомментаторов 90-х, банк готовых фраз ElevenLabs |
| Управление | ПК: клавиатура и геймпад. Планшет: виртуальный стик + 2–3 кнопки |
| Имена | Реальные составы 96–99 для себя и друзей. При публичном релизе — схема PES: Роналдо→Роландо, вымышленные эмблемы |

## Геймплейный камертон (железное правило)

**Реализм и оборона важнее зрелищности.** Типичный счёт матча — 1:0, 2:1, максимум 3:2. Разгром 7:3 = баг баланса, а не «весело получилось». Каждую правку геймплея проверяем вопросом: **«не стали ли голы слишком дешёвыми?»**

Четыре рычага низких счетов (из ресёрча, [подробно](База-знаний/Исследования/03-Футбольный-AI.md)):
1. Шум в ударах — точность как настраиваемый параметр, дальние удары редко летят в раму.
2. Вероятностная модель гола (мини-xG): шанс = дистанция + угол + давление защитника + реакция кипера.
3. Компактная оборона — в защите вся команда сжимается к линии мяча, форварда встречают 2–3 тела.
4. Перехват передач — защитники перекрывают линии паса, атаки обрываются до удара.

## Технологический фундамент (из ресёрча 17.07.2026)

- **Three.js через CDN** (importmap, ES-модули, без сборщиков) — проверенный на «Новоселье» паттерн, работает на ПК и планшете. UE5 отброшен: веб-экспорта у него нет, а нативная сборка на iPad = Apple Developer $99/год и TestFlight. [Полное сравнение](База-знаний/Исследования/01-Выбор-движка.md).
- **Рендер в низком разрешении (~320×240) с апскейлом** — это одновременно и стиль PS1, и главная экономия FPS на планшете.
- **CRT одним проходом** (scanlines + кривизна + виньетка, библиотека pmndrs/postprocessing) — каждый лишний проход бьёт по планшетному GPU.
- **Игроки и стадион — из примитивов кодом** (кубики, как настоящие игры 96–98): ноль лицензионных проблем, ноль мегабайт моделей. Blender держим в резерве для стадиона/кубков, если захочется деталей.
- **Газон** — текстура с полосами покоса + шейдерные нюансы. 3D-трава инстансингом — только если останется запас FPS.
- **AI — порт «Simple Soccer»** Мэта Бакленда: тренер (стейт-машина команды) → голова игрока (стейт-машина роли) → ноги (steering behaviors). Есть готовый JS-порт как образец.
- **Звук** — Howler.js, файлы MP3 (OGG не играет в Safari). Комментатор: ~120–150 целых фраз ElevenLabs (Eleven v3, аудио-теги [shouts] для «ГО-О-ОЛ!»), без склейки фамилий — на весь банк хватит месяца подписки Creator (~$11).
- **Составы** — JSON из английской Википедии (squad pages ЧМ-98/Евро-96: номер, позиция, возраст, клуб).
- **Раздача друзьям** — GitHub Pages: git push → через минуту обновлённая игра по ссылке, на любом устройстве.

## План фаз

### Фаза 0 — «Трансляция без футбола» (фундамент)
Цель: открываешь ссылку — и это уже выглядит как телетрансляция 98-го, хоть на поле никого нет.
- [ ] Скелет проекта: index.html + ES-модули, локальный запуск
- [ ] Поле с разметкой, воротами и полосатым газоном, ТВ-камера
- [ ] Рендер в низком разрешении + CRT-проход поверх
- [ ] Мяч с простой физикой (катится, отскакивает)
- [ ] Проверка на iPad: FPS, тач, звук по первому касанию
- [ ] Git-репозиторий + деплой на GitHub Pages

### Фаза 1 — «Один игрок и мяч»
Цель: бегать и бить приятно — «чувство мяча» решает всё, полируем до кайфа.
- [ ] Игрок из примитивов с процедурной анимацией бега
- [ ] Управление: клавиатура, геймпад (Gamepad API), виртуальный стик на таче
- [ ] Дриблинг (мяч липнет у ног), пас, удар с силой от удержания кнопки
- [ ] Камера следит за мячом как ТВ-оператор

### Фаза 2 — «22 игрока» (AI-каркас)
Цель: полноценный матч AI vs AI, в который уже можно вмешиваться.
- [ ] Порт Simple Soccer: стейт-машины команды и игрока, steering behaviors
- [ ] Формации как «домашние регионы» (4-4-2), роли защитник/полузащитник/нападающий
- [ ] Вратарь (держится между мячом и воротами, выходит на перехват)
- [ ] Переключение управляемого игрока, ближний к мячу
- [ ] Голы, ауты, угловые, вбрасывания; табло и таймер
- [ ] Все константы — в config.js (наш Params.ini)

### Фаза 3 — «Защита и баланс» (сердце проекта)
Цель: средний счёт 1:0–3:2 в автосимуляции 100 матчей AI vs AI.
- [ ] Компактная оборона: регионы сжимаются к линии мяча
- [ ] Персональная опека + прессинг владеющего мячом
- [ ] Перекрытие линий передач, перехваты по траектории
- [ ] Вероятностная модель удара (мини-xG) + активный вратарь
- [ ] Офсайды
- [ ] Скрипт автосимуляции матчей для замера баланса

### Фаза 4 — «Атмосфера 90-х»
Цель: закрываешь глаза — 1998-й.
- [ ] Комментатор: тексты фраз (скилл elevenlabs-v3-writer) → генерация банка → интеграция
- [ ] Звуки: стадион, реакция трибун на моменты, свисток
- [ ] Формы 8 сборных, меню и заставки в эстетике телеграфики 90-х
- [ ] Повтор гола (replay с другой камеры)

### Фаза 5 — «Составы и друзья»
Цель: друзья играют по ссылке и просят добавить их любимую сборную.
- [ ] Составы 8 сборных из Википедии → data/teams/*.json (имя, номер, позиция, характеристики)
- [ ] Усталость и замены; фолы, карточки, штрафные, пенальти
- [ ] Hot-seat вдвоём (два геймпада / клавиатура на двоих)
- [ ] Мини-турнир плей-офф на 8 команд
- [ ] Ссылка друзьям + сбор обратной связи

## Что сознательно НЕ делаем сейчас
Онлайн, скелетные анимации из Mixamo (стартуем с процедурных на примитивах), 3D-трава, лицензии, второй язык комментатора. Каждый пункт можно вернуть, когда текущие фазы дожаты.
