# Грабли и приёмы — «Футбол 98»

Рабочий журнал: **проблема → причина → решение.** Пополняется по ходу разработки. Стартовый набор — из ресёрча 17.07.2026 и опыта «Новоселья», по мере проверки на практике помечаем ✅.

## Звук
- **OGG не играет в Safari.** Все звуки и клипы комментатора — только MP3 (mono 64 kbps для голоса достаточно).
- **iOS: звук стартует только после первого касания экрана.** Автоплей заблокирован системой. Howler.js разблокирует аудио по первому тапу сам — но проверить на реальном iPad.

## Three.js и рендер
- **Современный Three.js — только ES-модули.** Старый `<script src=...>` не работает: нужен `importmap` + CDN (jsdelivr/unpkg). Держать локальную копию three.module.js на случай проблем с CDN.
- ✅ **Three.js — это ДВА файла (проверено 17.07.2026).** С версии ~r167 сборка разделена: `three.module.js` внутри импортирует `./three.core.js`. Скачал только первый — весь граф модулей молча падает с «Failed to fetch dynamically imported module». Качать оба в `vendor/`.
- ✅ **Ошибки модулей — молчаливые.** Упавший `<script type="module">` не всегда виден в консоли. В index.html стоит ловушка `window.addEventListener('error', ...)`, которая выводит сбой прямо на экран в #hint — не удалять её.
- **CRT-постобработка дорогая на планшете.** Каждый полноэкранный проход бьёт по мобильному GPU. Решение: ОДИН объединённый проход (scanlines+виньетка+кривизна) + рендер сцены в ~320×240 — что заодно и есть стиль PS1.
- **WebGL-контекст на iOS может «теряться»** (страница в фоне, нехватка памяти). Нужен обработчик `webglcontextlost`/`webglcontextrestored`, иначе — чёрный экран без ошибок.

## Рендер и муар
- ✅ **Разметка бьётся в пунктир и мигает (грабля от Олега, 17.07.2026).** Причина: тонкие линии (1 px в текстуре) + NearestFilter при уменьшении = муар. Лечение: у текстур поля/трибун `minFilter: LinearMipmapLinearFilter` + `anisotropy: 4` (вблизи можно оставить Nearest — PS1-пиксели не страдают), линии разметки рисовать толще реальных (0.3 м).
- ✅ **Превью-вкладка Claude может быть «скрытой» (document.hidden=true) — тогда requestAnimationFrame спит и игра выглядит намертво зависшей.** Это НЕ баг игры: на реальном устройстве всё работает. Проверка логики в таком состоянии — синхронный прогон `DBG.input/player/ball.update()` + ручной `DBG.crt.render()` через вставленный в страницу script.

## Браузер и кэш
- **Негативный кэш 404 (грабля из «Новоселья»).** Если код запрашивал файл, которого ещё не было, браузер запоминает 404. После добавления файла — жёсткое обновление (Ctrl+Shift+R; на планшете — очистить кэш/потянуть страницу), иначе файл «не появится».
- ✅ **`python3 -m http.server` кэширует насмерть (проверено 17.07.2026).** Отдаёт 304, браузер держит старые модули — после правки config.js страница гоняла старую версию. Лечение: свой `serve.py` с заголовком `Cache-Control: no-store` — запускать локалку только через него.
- ✅ **Отладка через браузер-панель Claude: JS-инструмент живёт в «изолированном мире».** `window.*` страницы оттуда не видно (window.DBG «нет», хотя он есть), а dynamic import не видит importmap — можно час ловить фантомные ошибки. Надёжный способ достучаться до мира страницы: вставить в DOM настоящий `<script>` элемент, который пишет результат в видимый элемент (#hint).

## Git и раздача
- **GitHub Pages не отдаёт файлы из Git LFS.** LFS в этом проекте не использовать; большие ассеты просто жать (наш бюджет и так маленький — примитивы + mp3).
- **Пути только относительные** (`./textures/...`), потому что на Pages сайт живёт в подпапке `/имя-репо/` — абсолютные пути `/textures/...` сломаются.

## Атмосфера и ассеты
- **FX-библиотека Олега** — `~/Documents/Рабочее/Ютуб/Клипы /Пыль на кассете/FX-Library/overlays/video/Loop (оверлеи)/` (47 луп-оверлеев). Для ТВ-пресетов пригодятся: `VHS-01/02/03`, `crt-scanlines`, `film-grain`, `vhs-head-switch`, `camcorder-datecode`, `tape-unspool`. Это 4K mp4 — перед использованием в игре пережать в ~640×360 low-bitrate и подмешивать `<video>` поверх канваса. Пока CRT чисто шейдерный (легче для планшета).
- **PS1-приём: свет «запечён».** Газон и мяч — MeshBasicMaterial (без расчёта света): ярко, дёшево и аутентично. Лампы/Lambert — только для объёмных объектов (ворота, трибуны).

## Геймплей
- **Разгромные счета в аркадах** берутся из трёх вещей: удары всегда точные, вратарь пассивный, оборона не сжимает пространство. Лечится параметрами в config.js: точность удара, вероятностная модель гола, компактность обороны, перехват пасов. Баланс проверять автосимуляцией 100 матчей, не «на глаз».
