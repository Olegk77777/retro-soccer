# Грабли и приёмы — «Футбол 98»

Рабочий журнал: **проблема → причина → решение.** Пополняется по ходу разработки. Стартовый набор — из ресёрча 17.07.2026 и опыта «Новоселья», по мере проверки на практике помечаем ✅.

## Звук
- **OGG не играет в Safari.** Все звуки и клипы комментатора — только MP3 (mono 64 kbps для голоса достаточно).
- **iOS: звук стартует только после первого касания экрана.** Автоплей заблокирован системой. Howler.js разблокирует аудио по первому тапу сам — но проверить на реальном iPad.

## Three.js и рендер
- **Современный Three.js — только ES-модули.** Старый `<script src=...>` не работает: нужен `importmap` + CDN (jsdelivr/unpkg). Держать локальную копию three.module.js на случай проблем с CDN.
- **CRT-постобработка дорогая на планшете.** Каждый полноэкранный проход бьёт по мобильному GPU. Решение: ОДИН объединённый проход (scanlines+виньетка+кривизна) + рендер сцены в ~320×240 — что заодно и есть стиль PS1.
- **WebGL-контекст на iOS может «теряться»** (страница в фоне, нехватка памяти). Нужен обработчик `webglcontextlost`/`webglcontextrestored`, иначе — чёрный экран без ошибок.

## Браузер и кэш
- **Негативный кэш 404 (грабля из «Новоселья»).** Если код запрашивал файл, которого ещё не было, браузер запоминает 404. После добавления файла — жёсткое обновление (Ctrl+Shift+R; на планшете — очистить кэш/потянуть страницу), иначе файл «не появится».

## Git и раздача
- **GitHub Pages не отдаёт файлы из Git LFS.** LFS в этом проекте не использовать; большие ассеты просто жать (наш бюджет и так маленький — примитивы + mp3).
- **Пути только относительные** (`./textures/...`), потому что на Pages сайт живёт в подпапке `/имя-репо/` — абсолютные пути `/textures/...` сломаются.

## Геймплей
- **Разгромные счета в аркадах** берутся из трёх вещей: удары всегда точные, вратарь пассивный, оборона не сжимает пространство. Лечится параметрами в config.js: точность удара, вероятностная модель гола, компактность обороны, перехват пасов. Баланс проверять автосимуляцией 100 матчей, не «на глаз».
