## Сводка ресёрча: браузерный ретро-футбол (эстетика 90-х)

### 1. Рекомендованный стек

**Three.js (библиотека 3D-графики для браузера) через CDN (загрузка библиотеки с чужого сервера, без установки)** — один HTML-файл плюс ES-модули (родная система разбиения кода на файлы, сборщик не нужен). Это единственный вариант, который повторяет уже проверенный тобой паттерн, работает на iPad и Android без серверных настроек, а низкополигональный футбол 11×11 в браузере планшета — доказанная реальность (Soccer Skills на Poki). Дополняем: постобработка (наложение эффектов на готовый кадр) — библиотека pmndrs/postprocessing, она сшивает все CRT-эффекты в один проход; игроки — из простых кубиков кодом; AI — порт «Simple Soccer» Бакленда; составы — JSON из Википедии; комментатор — заранее записанные клипы ElevenLabs; раздача — GitHub Pages. Godot и Babylon.js — запасные, UE5 отпадает.

### 2. Ключевые решения по темам

**Движок.** Three.js с рендером сцены в низком разрешении (~320×240) — это и стиль PS1, и главная экономия производительности на планшете. Godot глючит со звуком в iOS Safari, UE5 официально в браузер не экспортируется.

**Ассеты.** Игроков собрать из примитивов (кубики-тело, кубики-ноги) кодом — так и выглядели реальные футболисты в играх 96–98, плюс ноль проблем с лицензиями и весом. Стадион и мяч — тоже кодом; шрифты табло — Press Start 2P и Jersey 10 с Google Fonts; звуки — Freesound с фильтром CC0 (лицензия «делай что хочешь»).

**AI игроков.** Портировать «Simple Soccer» Мэта Бакленда — двухуровневые стейт-машины (наборы состояний «команда атакует/обороняется» и «игрок ждёт/бежит/бьёт») плюс steering behaviors (плавное рулевое движение к цели). Есть готовый JS-порт (phaser-simple-soccer), логика переносится почти дословно. Низкие счета 1:0–3:2 дают: шум в ударах, вероятностная модель гола, компактная оборона и перехват пасов — все константы в один конфиг для баланса.

**PS1-рендер и CRT.** Вершинный снаппинг (дрожание вершин как на PS1) и affine-текстуры — в материалах, а весь CRT (полосы развёртки, кривизна, виньетка) — одним объединённым проходом, потому что каждый лишний проход бьёт по планшетному GPU. Газон — просто текстура с полосами покоса, никакой 3D-травы.

**Составы и юридика.** Данные брать из английской Википедии (страницы squads ЧМ-98 и Евро-96 — номер, позиция, возраст, клуб). Имена — изменённые, но узнаваемые (Роналдо→Роландо), клубы — вымышленные: это путь PES, который снимает риск претензий. Урок дела Keller против EA: даже без имён полное совпадение атрибутов — тонкий лёд.

**Комментатор.** Банк из ~120–150 целых фраз по категориям (гол — 8–10 вариантов, сейв — 5–6), без склейки фамилий — она и была источником багов FIFA 97. Модель Eleven v3 с аудио-тегами вроде [shouts] для «ГО-О-ОЛ!»; весь банк — один месяц подписки Creator (~$11). Формат MP3 mono 64 kbps — OGG не играет в Safari.

**Раздача.** GitHub Pages в режиме «Deploy from a branch»: git push — и через минуту сайт обновлён, ноль настройки. Пути к файлам — только относительные (`./textures/...`), Git LFS не использовать (Pages его не отдаёт). Запасной вариант — itch.io Draft с секретной ссылкой.

### 4. Главные риски и как их снять

**AI — самое сложное, не графика.** Логика 22 игроков — основной объём работы. Снятие: не изобретать, а портировать готовый Simple Soccer, все параметры баланса — в отдельный конфиг.

**Разрастание кода.** 5–10 тысяч строк в одном файле станут неуправляемыми. Снятие: с первого дня разбивать на ES-модули (сборщик для этого не нужен).

**iOS Safari.** Звук стартует только по касанию, память жёстче, контекст WebGL может теряться. Снятие: разблокировка аудио по первому тапу (Howler.js делает сам), обработчик потери контекста, тест на реальном iPad с первых недель.

**Юридика при публикации.** Реальные имена и эмблемы — риск письма с требованием убрать игру. Снятие: схема PES (изменённые имена, вымышленные клубы) сразу, реальный датасет — отдельным пользовательским файлом.

**Разгромные счета убьют «футбольность».** Снятие: точность ударов и активный вратарь как настраиваемые параметры, балансировка через конфиг без правки кода.
