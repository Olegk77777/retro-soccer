# Отчёт: выбор движка для браузерного «ретро-футбола» (эстетика 90-х, CRT, ПК + планшеты)

Дата исследования: 17 июля 2026.

## Главный вопрос: потянет ли браузер планшета low-poly футбол 11×11?

**Да, это доказано существующими играми.** На Poki работает [Soccer Skills Champions League](https://poki.com/en/g/soccer-skills-champions-league) (Radical Play) — «3D fast-paced soccer game that has 11-versus-11 matches», официально поддерживаемые устройства: «desktop, phone and tablet». Там же — [Super Liquid Soccer](https://poki.com/en/g/super-liquid-soccer) (3D, 7×7) и целый раздел [футбольных игр на CrazyGames](https://www.crazygames.com/t/soccer). 22 low-poly персонажа + стадион — нагрузка уровня 2015 года; современный iPad с WebGL2 (а с осени 2025 и WebGPU) справляется с запасом. Узкое место — не число игроков, а дисциплина: мало draw calls, инстансинг, простые материалы, один полноэкранный CRT-шейдер (не многопроходный).

## Варианты

### (а) Three.js — чистый JS через CDN
- Актуальная версия: npm-пакет `three` ~0.185.x, релизы выходят ежемесячно ([releases](https://github.com/mrdoob/three.js/releases), [npm](https://www.npmjs.com/package/three)). В 2025–26 добавился зрелый WebGPU-рендерер (TSL), но WebGL2-путь остаётся дефолтным и идеален для ретро-стиля ([обзор изменений 2026](https://www.utsubo.com/blog/threejs-2026-what-changed)).
- Нюанс: современные версии распространяются только как ES-модули — нужен `importmap` + CDN (jsdelivr/unpkg), это чуть иной шаблон, чем старые `<script src>`.
- CRT-фильтр — штатный EffectComposer/ShaderPass, готовых CRT-шейдеров много.
- Работает на iPad/Android-браузерах без каких-либо заголовков сервера и без сборщиков. Паттерн у Олега уже проверен на PS1-игре.

### (б) Godot 4.x веб-экспорт
- Godot 4.0–4.2 на iOS Safari **не работал** (SharedArrayBuffer + баги WebGL2). В Godot 4.3 (2024) вернули **однопоточный экспорт**, и «these issues fortunately disappear» на macOS/iOS — официальный [progress report](https://godotengine.org/article/progress-report-web-export-in-4-3/).
- Многопоточный экспорт требует SharedArrayBuffer, то есть заголовков COOP/COEP на сервере — [частая причина «работает локально, ломается на хостинге»](https://www.strayspark.studio/blog/godot-46-export-web-android-ios-guide).
- Актуальные версии: 4.5.2 / 4.6 ([Godot Mobile update, апрель 2026](https://godotengine.org/article/godot-mobile-update-apr-2026/)). Но на iOS Safari остаются шероховатости: например, [issue #107390](https://github.com/godotengine/godot/issues/107390) — веб-игра со звуком крашится/перезагружается через несколько минут в iOS Safari/Chrome.
- Вес пустого веб-билда: ~40 МБ (≈5 МБ Brotli) — из того же progress report. Итог: работоспособно, но iOS Safari — исторически самая хрупкая платформа Godot-web.

### (в) Babylon.js
- Версия 8.0 — [март 2025](https://blogs.windows.com/windowsdeveloper/2025/03/27/announcing-babylon-js-8-0/): нативный WebGPU (WGSL), движок стал в 2 раза меньше, IBL-тени, area lights. Полноценный «движок в браузере»: физика, аудио, GUI из коробки. Тоже ставится через CDN без сборщика. По [сравнениям](https://blog.logrocket.com/three-js-vs-babylon-js/) чуть тяжелее и «умнее», чем Three.js; для low-poly ретро его богатый PBR-пайплайн просто не нужен.

### (г) PlayCanvas
- Движок open source (MIT), runtime 1–2 МБ, самый зрелый WebGPU среди веб-движков ([playcanvas.com](https://playcanvas.com/), [сравнение движков 2026](https://app.cinevva.com/guides/web-game-engines-comparison)). Но главная ценность — облачный редактор (freemium, проприетарный): рабочий процесс завязан на браузерный UI, что плохо стыкуется с паттерном «Claude Code пишет файлы локально». Использовать только engine-as-library можно, но тогда это «Three.js с другим API» без явных преимуществ.

### (д) Unreal Engine 5
- **Официального веб-экспорта нет**: HTML5 удалён ещё в UE 4.24, официальный путь в браузер — Pixel Streaming (рендер на сервере) ([The New Stack](https://thenewstack.io/a-new-tool-for-unreal-engine-developers-to-export-to-the-web/), [форум Epic](https://forums.unrealengine.com/t/new-platform-and-tools-for-html5-exports/260090)).
- Рабочий сторонний форк существует: Wonder Interactive / [SimplyStream](https://simplystream.com/) — UE 5.6/5.8 → WebAssembly + WebGPU, с асset-стримингом; инструменты бесплатны, но хостинг игры — по подписке на их платформе ([GamesBeat](https://gamesbeat.com/simplystream-unlocks-web-compatibility-for-unreal-engine-5-enabling-high-quality-webgpu-gaming-in-the-browser/), [форум](https://forums.unrealengine.com/t/webgpu-for-unreal-engine-5-5-6-and-5-7-support/2693960)).
- Вес: ещё минимальный UE4-HTML5-билд весил [~155 МБ](https://forums.unrealengine.com/development-discussion/html5-development/75761-why-is-the-size-of-even-the-most-minimal-html5-project-~-155-mb); стриминг ассетов это маскирует, но не для планшетного железа. WebGPU в Safari появился только с iOS/iPadOS 26 (сентябрь 2025, [WebKit blog](https://webkit.org/blog/16993/news-from-wwdc25-web-technology-coming-this-fall-in-safari-26-beta/)) — старые iPad отпадают.
- Нативная сборка на iPad: Xcode + аккаунт Apple Developer ($99/год) для TestFlight; бесплатный профиль ставит игру лишь на свой iPad на 7 дней. Для хобби-проекта — неоправданно.

## Вердикт

**Three.js через CDN — вариант (а).** Он единственный полностью повторяет уже сработавший паттерн Олега (один HTML-файл, ES-модули с CDN, ноль сборки), покрывает обе платформы без серверных заголовков и нативных сборок, а low-poly футбол 11×11 на планшете — доказанная реальность (Soccer Skills). Babylon.js — запасной вариант, если захочется встроенной физики. Godot — только если проект перерастёт в «настоящий» геймдев; UE5 отбрасываем.

## Риски пути Three.js

1. **Нет движка — всё руками**: игровой цикл, ИИ 22 игроков, столкновения. Для аркадного футбола хватит простой математики (круги/капсулы), полноценный физдвижок не нужен — но это самая сложная часть кода, не рендер.
2. **Рост одного файла**: 5–10 тыс. строк в index.html станут неуправляемыми; стоит сразу разбить на ES-модули (это не требует сборщика).
3. **iOS Safari**: звук стартует только по тапу пользователя; лимиты памяти жёстче десктопа; возможна потеря WebGL-контекста — нужен обработчик.
4. **CRT-шейдер на слабых планшетах**: полноэкранный постпроцессинг удваивает нагрузку на филлрейт — делать один комбинированный проход (scanlines + виньетка + аберрация) и рендерить сцену в пониженном разрешении (что ещё и аутентично для 90-х).
5. **Зависимость от CDN**: без интернета игра не загрузится; решается локальной копией three.module.js рядом с проектом.
6. **Права на реальные составы 1996–99** — для хобби некоммерчески риск низкий, но при публикации на порталах (CrazyGames/Poki) имена реальных игроков и клубов могут стать проблемой лицензирования.
